Trusting Docker Official Images
===============================

When the daemon is configured with
```json
{
  "content-trust": {
    "trust-pinning": {
      "official-library-images": true
    }
  }
}
```
a list of trusted root keys which have been hardcoded into the daemon binary are merged into the user-provided trust pinning configuration. This hardcoded list is defined in `doi_root_keys.go`, which is generated by `gen-doi.sh` from `doi-keys.txt`.

Refreshing `doi-keys.txt`
-------------------------

The `doi-keys.txt` file was generated with

```console
$ ./gen-doi.sh > doi-keys.txt
```

and can be run again to refresh the list. The script looks up the root keys used for signing all actively-maintained official images (defined as having a spec file in the HEAD of the default branch of https://github.com/docker-library/official-images) along with some hand-picked deprecated images from notary.docker.io.
